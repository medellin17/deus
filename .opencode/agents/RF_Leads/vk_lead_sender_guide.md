# Инструкция: Автоматическая рассылка лидов через VK Lead Sender

Инструмент **VK Lead Sender** автоматизирует отправку подготовленных сообщений (питчей) администраторам и владельцам бьюти-салонов и стоматологий ВКонтакте.

Инструмент находится по адресу:  
`C:\Users\gleb7\OneDrive\Рабочий стол\vibecoding\Projects\VkLeadSender`

---

## 🛠️ Подготовка и интеграция с базой лидов

Процесс обновления и отправки лидов полностью бесшовный:
1. Вы находите новые лиды и вносите их в скрипт базы `generate_rf_docs.py` (папка `RF_Leads`) со статусом `"Не начато"`.
2. Запускаете `python generate_rf_docs.py`. Скрипт:
   * Фильтрует все лиды со статусом `"Не начато"`.
   * Генерирует для каждого лида индивидуальное сообщение (`OutreachText`) без упоминания скорости, разбитое на абзацы и с вашей подписью.
   * Экспортирует эти лиды в `leads.json`.
   * **Автоматически копирует** получившийся файл в папку рассыльщика по пути:  
     `..\Projects\VkLeadSender\leads\rf_leads.json`.

## 🚀 Быстрый старт рассылки

Перейдите в директорию проекта:
```powershell
cd "C:\Users\gleb7\OneDrive\Рабочий стол\vibecoding\Projects\VkLeadSender"
```

### 1. Авторизация (выполняется один раз)
Для работы скрипта нужен токен доступа VK (с правами на сообщения и группы). Запустите команду авторизации:
```bash
python vk_lead_sender.py --login
```
Скрипт предложит перейти на сайт [vkhost.github.io](https://vkhost.github.io/), получить токен (например, через VK Admin или Настройки) и вставить полученный URL или токен в консоль. Токен сохранится локально в `~/.vkleadsender/config.json`.

### 2. Тестовый запуск (Dry-Run)
Перед реальной отправкой обязательно проверьте, кому и какие сообщения уйдут. Это не отправляет реальные сообщения, а лишь показывает планируемые действия:
```bash
python vk_lead_sender.py --dry-run --verbose
```

### 3. Запуск реальной рассылки
Чтобы VK не заблокировал аккаунт за спам, рассылку необходимо растягивать во времени. По умолчанию рассылка идет с интервалами, но вы можете явно указать время (в минутах), на которое нужно растянуть отправку всех сообщений из базы:
```bash
# Растянуть отправку всех подготовленных лидов на 60 минут
python vk_lead_sender.py --interval 60
```

### 4. Продолжение после паузы/сбоя
Если рассылка была прервана или вы хотите продолжить отправку недоотправленным лидам:
```bash
python vk_lead_sender.py --resume
```
Состояние отправки кэшируется в файле `state.json`.

## 📋 Таблица основных команд и флагов

| Флаг | Описание | Пример использования |
|------|----------|----------------------|
| `--login` | Первичная авторизация (получение токена VK) | `python vk_lead_sender.py --login` |
| `--reset-auth` | Сбросить текущий сохраненный токен | `python vk_lead_sender.py --reset-auth` |
| `--dry-run` | Тестовый прогон без реальной отправки сообщений | `python vk_lead_sender.py --dry-run` |
| `--interval N` | Время в минутах, на которое растягивается вся сессия рассылки | `python vk_lead_sender.py --interval 45` |
| `--resume` | Продолжить рассылку с места остановки | `python vk_lead_sender.py --resume` |
| `--reset-state` | Сбросить кэш отправленных лидов (начать заново) | `python vk_lead_sender.py --reset-state` |
| `--mark-done N` | Вручную пометить N-й лид как отправленный (пропустить его) | `python vk_lead_sender.py --mark-done 3` |
| `--verbose` | Детальный лог работы (вывод текста сообщений и деталей API) | `python vk_lead_sender.py --dry-run --verbose` |

## ⚙️ Как работает пайплайн обработки лида внутри скрипта

Когда запускается рассылка, для каждого лида из `rf_leads.json` происходит следующее:
1. **Анализ ссылки:** Скрипт парсит контакты лида, находя ссылки на группы ВК (`vk.com/...`).
2. **Поиск администраторов:** Через VK API (`groups.getMembers` с фильтром `managers`) скрипт запрашивает список руководителей/модераторов этой группы.
3. **Отправка сообщения:** 
   * Если руководители найдены и у них открыта личка — сообщение отправляется им в ЛС.
   * Если ЛС закрыто или руководители скрыты — скрипт пытается отправить сообщение в ЛС самого сообщества (если там включены сообщения).
4. **Сбор телефонов:** Все номера телефонов лидов выгружаются в файл `leads/phone_leads.txt`. Его можно использовать для рассылки в WhatsApp или прозвона.
5. **Ручной контроль:** Если у лида нет ВК-группы или не удалось отправить сообщение автоматически, он помечается статусом `manual_check`, чтобы вы могли обработать его вручную.

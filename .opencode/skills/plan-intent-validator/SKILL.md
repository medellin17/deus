# Проверка соответствия плана исходной задаче

Выполнить 5-проходную проверку: извлечь intent из задачи, проверить покрытие, scope creep, интерпретацию и критерии готовности. Итог — вердикт `ALIGNED` / `NEEDS_REVISION`.

**Шаблонные переменные**: `{{task}}` (исходная задача), `{{PLAN.md}}` (план).

> [!IMPORTANT]
> Не писать код. Не дорабатывать план. Только анализ и вердикт.

**Guard**: если `{{task}}` или `{{PLAN.md}}` пусты — вернуть `NEEDS_REVISION` с указанием причины.

---

## Проход 1. Извлечение intent

Выделить из задачи:
1. **Главная цель** — одним предложением, что должно получиться.
2. **Обязательные требования** — без чего задача не считается выполненной.
3. **Явные ограничения** — технологии, scope, что исключено.
4. **Неявные ожидания** — качество, безопасность, тесты, документация из контекста.

---

## Проход 2. Полнота покрытия

Сравнить каждое обязательное требование из Прохода 1 с `PLAN.md`. Для каждого:

- **Покрыто** — где и как.
- **Покрыто частично** — что упущено.
- **Не покрыто** — требование отсутствует.

Если все покрыты — явно: «Все требования покрыты».

---

## Проход 3. Scope creep

Выявить в плане элементы, которые:
- не запрашивались в задаче;
- выходят за явные ограничения;
- добавляют функциональность/слои/зависимости без прямой необходимости.

Для каждого: что добавлено, почему вне рамок, влияние на объём/сроки.

Если нет — явно: «Scope creep не выявлен».

---

## Проход 4. Интерпретация

Проверить, не переформулирована ли задача под другую проблему:
- "Зачем?" в плане соответствует исходной боли?
- "Что?" соответствует заявленному результату?
- Нет подмены цели?

Если нет — явно: «Искажение интерпретации не выявлено».

---

## Проход 5. Критерии готовности

Для каждой секции "Как проверить?" в плане:
- Критерий проверяет то, что описано в "Что?" этой же карточки?
- Можно ли однозначно доказать выполнение (конкретные сценарии, метрики)?
- Нет проверки, замыкающейся на себя ("код написан" вместо "функция работает")?

**Guard**: если секций >10 — проверить 5 наиболее рискованных по твоей оценке.

Найти минимум 1 критерий, который не проверяет заявленное поведение. Если все проверяют — явно: «Все критерии проверяют заявленное поведение».

---

## Итоговый вердикт

### `ALIGNED`
Только если одновременно:
- Все обязательные требования покрыты
- Scope creep не выявлен
- Интерпретация не искажена
- Критерии проверяют заявленное поведение

### `NEEDS_REVISION`
При любом расхождении.

**Формат вывода**:

```
Вердикт: ALIGNED / NEEDS_REVISION

Полнота покрытия:
- [✓/~] Требование: ... → Статус

Scope creep:
- ... (или "Не выявлен")

Искажение интерпретации:
- ... (или "Не выявлено")

Проблемы с критериями:
- ... (или "Не выявлены")

Рекомендация:
- ALIGNED: "План можно утверждать."
- NEEDS_REVISION: список для plan-refiner.
```

> [!WARNING]
> Не занижать severity ради вердикта. Если есть обоснованное расхождение — `NEEDS_REVISION`.
